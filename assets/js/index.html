Comandos principales

Configuración

  • git config --global user.name "gaby28894178" # configurar el nombre de usuario
  • git config --global user.email gabgabrielligabriel@gmail.com # configurar el correo electrónico
  • git config --global core.editor "code --wait" # configurar el editor de texto
  • git config --global core.autocrlf # [true|input] # configurar el formato de los archivos
  • git config --global init.defaultBranch main # Asegura que los repositorios recién creados usen el nombre main como rama principal
  • git config --global -e # editar el archivo de configuración global
  • git config --list # ver la configuración global
  • git config -h # ver la ayuda de la configuración

Repositorio Local

  • git init # crea un nuevo repositorio en el directorio actual
  • git status # muestra el estado del repositorio
  • git status --short # muestra el estado del repositorio resumido
  • git add «archivo» # agregar archivos al área de preparación
  • git add . # agregar todos los archivos da prioridad al archivo .gitignore
  • git commit -m "descripción breve" # crear un nuevo commit con un mensaje
  • git commit -am "descripción breve" # agregar y crear un nuevo commit con un mensaje
  • git log # ver el historial de commits
  • git log --oneline # ver el historial de commits en una sola línea

Repositorio Remoto

  • git remote add (origin(nombre-corto)) (url) # agrega un nuevo repositorio Git remoto como un nombre corto al que puede hacer referencia fácilmente.
  • git push (remote) (branch) # se utiliza para enviar los cambios confirmados (commits) desde la rama local de un repositorio Git a la rama correspondiente en un repositorio remoto. Si la rama remota no existe aún, Git la creará automáticamente.
  • git push --all (remote) # envia los cambios confirmados a todas las ramas locales al repositorio remoto.
  • git push # envia los cambios confirmados sobre el repositorio remoto al que te encuentras
  • git pull # Descarga las actualizaciones del repositorio remoto y fusiona los cambios descargados con los cambios locales en una sola operación.

Ramas

  • git branch # ver las ramas
  • git switch -c "rama" # crear una nueva rama y cambiar a ella
  • git branch -d "rama" # eliminar una rama
  • git branch -D "rama" # eliminar una rama sin aceptar cambios
  • git branch -m "rama" "new-name" # renombrar una rama
  • git switch "rama" # cambiar de rama
  • git merge "rama" # fusionar una rama con la rama actual

Estados de git

Cuando se trabaja con Git, hay tres estados principales en los que pueden estar los archivos en un repositorio:

  • (M) Modificado (Modified) : Este estado indica que el archivo ha sido modificado desde la última confirmación. Es decir, el archivo ha sido editado, pero aún no ha sido agregado al área de preparación.
  • (A) Agregado (Added) : Este estado indica que el archivo ha sido agregado al área de preparación y está listo para ser confirmado en el próximo commit. Los cambios realizados en este archivo se han confirmado a nivel local, pero aún no se han enviado al repositorio remoto.
  • (U) Sin seguimiento (untracked) : Este estado indica que el archivo no se encuentra en el índice de Git y no ha sido confirmado. Puede ocurrir cuando un archivo ha sido creado recientemente en el directorio de trabajo, pero aún no ha sido agregado al repositorio.
  • U: Actualizado (Updated) Además de estos estados principales, también hay otros estados secundarios que se pueden mostrar en algunos casos, como "C" para conflictos, "D" para eliminados, "R" para renombrados y "I" para ignorados. Estos estados se pueden verificar en el terminal mediante el comando "git status".
